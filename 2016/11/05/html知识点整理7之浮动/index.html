html&css知识点整理7---浮动

发表于 2016-11-05   |   分类于 html&css知识点整理   |  

基础知识

浮动：浮动可以理解为让某个div元素脱离标准流，漂浮在标准流之上。浮动的框可以左右移动，直到它的外边缘碰到包含框或另一个浮动框的边缘。因为浮动框不在文档的普通流中，所以文档的普通流中的块框表现得就想浮动框不存在一样 。此外，浮动元素会生成一个块级框，而不论这个元素本身是什么。因此，如果让一个链接浮动，解释该链接本身是行内元素，通常会生成一个行内框，但只要它是浮动的，就会生成一个块级框，可以设置width、height属性。

（2）、第一个盒子向右浮动float:right

当把第一个盒子向右浮动时，它脱离文档流并向右移动，直到它的右边缘碰到包含框的右边缘。
（3）、第一个盒子向左浮动float:left


当把第一个盒子向左浮动时，它脱离文档流并向左移动，直到它的左边缘碰到包含框的左边缘。因为它不再处于文档流中，所以不占据空间，实际上覆盖住了框2，使框2从视图上消失。
（4）、三个盒子都浮动

如果把所有3个框都向左浮动，那么框1向左浮动直到碰到包含框，另外两个向左浮动直到碰到前一个浮动框。这里父元素设置了清除浮动，所以父元素高度没有塌陷。

如果包含框太窄，无法容纳水平排列的浮动元素，那么其他浮动块向下移动，直到有足够空间的地方。

如果浮动元素的高度不同，那么当它们向下移动时可能会被其他浮动元素“卡住”。

三个盒子都向右浮动，由于是右浮动，因此右边靠近页面边缘，所以右边是前，因此.box1在最右边。

浮动与行框

前面指出浮动会让元素脱离文档流，不影响不浮动元素。实际上并不完全如此，如果浮动的元素后面有一个文档流中元素，那么这个元素的框会表现的像浮动元素不存在，但是框的文本内容会受到浮动元素的影响，会移动以留出空间。用术语说就是浮动元素旁边的行框被缩短，从而给浮动元素流出空间，因而行框围绕浮动框。这个特性可以制作文本围绕图片的效果。

浮动对其他元素的影响

浮动元素对于浮动元素

元素浮动后如果前面有其他浮动元素，则会依次排在其后面，如果前面的浮动的元素已经占领一整行或排列不下，元素会跳到下一行。但是当浮动元素高度不同，那么当它们向下移动时可能会被其他浮动元素“卡住”。

浮动元素对于普通元素

  • 块级元素的背景和边框会被该元素遮挡，而内容会环绕浮动元素。

  • 内联元素不会被遮挡，而是会靠着浮动元素的外边距排列

浮动元素对于文字

浮动对于文字，会使得文字围绕着浮动元素。

浮动带来的影响

父元素高度塌陷
我们知道，一个块级元素如果没有设置height，其height是由子元素撑开的。对子元素使用了浮动之后，子元素会脱离标准文档流，也就是说，父级元素中没有内容可以撑开其高度，这样父级元素的height就会被忽略，这就是所谓的高度塌陷。

由上图可知，三个子元素都设置了浮动，因此父元素高度会塌陷。

clear属性

有的时候，我们不希望一些元素会被旁边的浮动元素影响到，这个时候就需要用到clear属性了。
clear属性：确保当前元素的左右两侧不会有浮动元素。
使用clear属性的时候要记住：clear只对元素本身的布局起作用。

清除浮动的方法

（1）、增加额外的div
这是最简单直接的方法，哪里有浮动元素，就在其父级元素的内容中增加一个（作为最后一个子元素）。

这样就会清除浮动元素带来的问题。
优点：简单直接，初学者常常使用的方法，也易于理解
缺点：增加额外的无意义标签，不利于语义化，每次清除都要添加额外的空标签，造成浪费
（2）、父级元素添加overflow:hidden
这个方法的关键在于触发了BFC，计算BFC的高度时，浮动元素也参与计算。

优点：代码量少，没有额外的标签
缺点：如果子元素超出父元素的范围，会造成超出的部分被隐藏
（3）、: after伪元素
after表示子元素的后面，通过它可以设置一个具有clear的元素，然后将其隐藏

这种方法的原理和第一个方法一样，就是生成一个元素来清除浮动，只是这个元素是“假的”。
优点：没有额外标签，综合起来算是比较好的方法
缺点：稍显复杂，但是理解其原理后也挺简单的
推荐使用这种方法。

浮动与定位同时作用于一个元素

（1）、相对定位遇上浮动
元素同时应用了position: relative（top / left / bottom / right）、float属性后，则元素先浮动到相应的位置，然后再根据（top / left / bottom / right）所设置的距离来发生偏移。此时可以给父元素清除浮动生效。

（2）、绝对定位遇上浮动
元素同时应用了position: absolute及float属性，则float失效。固定定位同样如此，float会失效。并且给父元素设置清除浮动也没有效果。


橘色盒子虽然设置了向右浮动，但是float:right失效了。
（3）、两个相邻元素，前一个设置定位，后一个设置浮动，则如果有重叠，定位元素会覆盖在浮动元素之上。

应用

（1）、文字围绕图片
（2）、横向导航菜单中的li通常设置浮动。
（3）、布局中经常用到浮动。

总结

（1）、float会造成元素脱离文档流，从而使浮动元素漂浮在正常流之上
（2）、浮动元素会生成一个块级框，而不论这个元素本身是什么
（3）、设置了浮动后，一定要记住给父元素清理浮动，否则会容易引发问题
（4）、浮动元素的后一个元素会围绕着浮动元素（典型运用是文字围绕图片）
