html和css知识点整理5---盒模型

发表于 2016-11-06   |   分类于 html和css知识点整理   |     |   阅读次数

盒模型概述

页面上的每个元素被看做一个矩形框，这个框由元素的内容、内边距、边框和外边距组成。
说明：
（1）、padding，border，margin都是可选的，默认值为0，但是浏览器会自行设置元素的margin和padding。
（2）、如果给元素设置背景，并且边框的颜色为透明，背景将应用于内容，内边距和边框组成的区域。
盒模型一共有两种模式，一种是标准模式，另一种就是怪异模式。
由于历史的原因，各个浏览器在对页面的渲染上存在差异，甚至同一浏览器在不同版本中，对页面的渲染也不同。在W3C标准出台以前，浏览器在对页面的渲染上没有统一规范，产生了差异(怪异模式)；由于W3C标准的推出，浏览器渲染页面有了统一的标准(标准模式)，这就是二者最简单的区别。

标准模式


在标准盒模型中，width和height是指内容区的宽度和高度。增加内边距、边框和外边距不会影响内容区域的尺寸，但是会增加元素框的总尺寸。

怪异模式（IE盒模型）


在怪异模式下，width/height 包括content尺寸＋padding＋border。

  • 早期IE6、IE7使用“IE盒模型”，后来更新了一次，更新后的IE6、IE7使用标准盒模型，IE8及以上版本使用标准盒模型
  • 没有DOCTYPE的情况下使用怪异模式，IE也使用“IE盒模型”

box-sizing属性

box-sizing有两个值：
当设置为box-sizing:content-box时，将采用标准模式解析计算，也是默认模式；
当设置为box-sizing:border-box时，将采用怪异模式解析计算；

块级元素和行内元素盒模型

块级元素：
（1）、会另起一行，
（2）、可以设置width，height，margin，padding，border属性
（3）、默认宽度是容器的100%
行内元素：
（1）、和其他元素在同一行内
（2）、高度和宽度就是内容的高度和宽度
（3）、可以设置margin属性，margin-left和margin-right属性，margin-top和margin-bottom属性不生效

左右margin生效，上下margin不起作用。margin-top和margin-bottom对内联元素（对行）的高度没有影响，并且由于边界效果(margin效果)是透明的，它也没有任何的视觉影响。
（4）、可以设置padding属性，左内边距应用到元素的开始处，右内边距应用到元素的最后，上下内边距有，但是行高不会变，如果设置背景色，背景色会延伸到上面和下面。

上图中，背景色延伸到上下padding，遮盖住了前面的行。

图中，虽然为行内元素设置了上下padding，但不会影响行高。
（5）、可以为行内元素设置border,对于左右边框，不仅边框可见，而且文本在旁边显示（即不会覆盖左右两边的文本），但上下边框对行高没有影响,且会覆盖上下文本。

（6）、这些个元素img|input|select|textarea|button|label虽然是内联元素，但border、padding、margin依旧可以影响到它的上下左右！

用margin还是用Padding

（1）、何时应当使用margin

  • 需要在border外侧添加空白时。
  • 空白处不需要背景（色）时。
  • 上下相连的两个盒子之间的空白，需要相互抵消时。如15px + 20px的margin，将得到20px的空白。

（2）、何时应当时用padding

  • 需要在border内测添加空白时。
  • 空白处需要背景（色）时。
  • 上下相连的两个盒子之间的空白，希望等于两者之和时。如15px + 20px的padding，将得到35px的空白。

个人认为：margin是用来隔开元素与元素的间距；padding是用来隔开元素与内容的间隔。margin用于布局分开元素使元素与元素互不相干；padding用于元素与内容之间的间隔，让内容（文字）与（包裹）元素之间有一段“呼吸距离”。

负margin的使用

（1）、当margin四个值都为正数值的话，那么margin按照正常逻辑同周围元素产生边距；
（2）、margin-top、margin-left为负值的时候，会把元素上移、左移，同时文档流中的位置也发生相应变化；
（3）、当margin-bottom、margin-right设为负值的时候，元素本身没有位置变化，后面的元素会上移、左移，因此margin-bottom、margin-right对自己没有影响，只会影响到下边、右边的元素；
（4）、如果一个元素没有设置宽度，并且为它设置负的margin-left、margin-right，则它的宽度会扩大。这是因为：父元素width=子元素width+padding+border+margin，如果margin为负值，则自然width会增大。


PS. 效果能实现的前提是元素的宽度不能设置为auto以外的值

外边距合并

兄弟元素外边距合并

外边距合并指的是，当两个垂直外边距相遇时，它们将形成一个外边距。合并后的外边距的高度等于两个发生合并的外边距的高度中的较大者。

如果想让上下外边距不合并，则可以给其中一个元素添加父元素，并另父元素形成一个BFC。

父子元素外边距合并

当一个元素包含在另一个元素中时（假设没有内边距或边框把外边距分隔开），它们的上和/或下外边距也会发生合并。

如果不想让二者的外边距合并，可以给父元素加上padding或是border。

元素自身上下外边距合并

如果一个元素没有内容、padding、border,则它自身的上下margin也可以合并。这种情况只要给该元素内容、padding、border即可。

注意：只有普通文档流中块框的垂直外边距才会发生外边距合并。行内框、浮动框或绝对定位之间的外边距不会合并。

总结

（1）、margin 属性可以应用于几乎所有的元素，除了表格显示类型（不包括 table-caption, table and inline-table）的元素，而且垂直外边距对非置换内联元素（non-replaced inline element）不起作用。
