html&css知识点整理3---css选择器

发表于 2016-10-23   |   分类于 html&css知识点整理   |  

选择器规则结构

每个规则都有两个基本部分：选择器和声明块。声明块由一个或多个声明组成，每个声明则是一个一个属性-值对，每个样式表有一系列规则组成。如规则左边所示，选择器定义了将影响文档中的哪些部分。规则右边包含声明块，它由一个或多个声明组成。

常用选择器分类

  1. 基础选择器
  2. 组合选择器
  3. 属性选择器
  4. 伪类选择器
  5. 伪元素选择器

基础选择器

选择器 含义
* 通用元素选择器，匹配页面任何元素（这也就决定了我们很少使用）
#id id选择器，匹配特定id的元素
.class 类选择器，匹配class包含(不是等于)特定类的元素
element 标签选择器，通常用于重置

组合选择器

选择器 含义
E,F 多元素选择器，同时匹配元素E和F
E F 后代选择器，用空格分隔，匹配E元素所有的后代（不只是子元素、子元素向下递归）元素F
E>F 子元素选择器，用>分隔，匹配E元素的所有直接子元素
E+F 直接相邻选择器，匹配E元素之后的相邻的同级元素F

属性选择器

选择器 含义
E[attr] 匹配所有具有属性attr的元素，div[id]就能取到所有有id属性的div
E[attr = value] 匹配属性attr值为value的元素，div[id=test],匹配id=test的div

伪类选择器

选择器 含义
E:hover 匹配鼠标悬停其上的E元素
E:link 匹配所有未被点击的链接
E:visited 匹配所有已被点击的链接
E:active 匹配鼠标标已经其上按下、还没有释放的E元素
E:focus 匹配获得当前拥有输入焦点的E元素
E:first-child 匹配元素E的第一个子元素
E:first-of-type 匹配父元素下使用同种标签的第一个子元素

Q:first-child和:first-of-type的作用和区别
:first-child 匹配的是某父元素的第一个子元素
:first-of-type匹配的是某父元素下相同类型子元素中的第一个，比如 p:first-of-type，就是指所有类型为p的子元素中的第一个。这里不再限制是第一个子元素了，只要是该类型元素的第一个就行了。

p:first-child 匹配到的是p元素,因为p元素是div的第一个子元素；
h1:first-child 匹配不到任何元素，因为在这里h1是div的第二个子元素，而不是第一个；
span:first-child 匹配不到任何元素，因为在这里两个span元素都不是div的第一个子元素；
p:first-of-type 匹配到的是p元素,因为p是div的所有类型为p的子元素中的第一个；
h1:first-of-type 匹配到的是h1元素，因为h1是div的所有类型为h1的子元素中的第一个；
span:first-of-type 匹配到的是第三个子元素span。这里div有两个为span的子元素，匹配到的是它们中的第一个。

.outer h1:first-child：会匹配到“我是大标题1”和“我是大标题2”，因为它们都是其父元素的第一个子元素。
.outer h2:first-child：不会匹配到任何元素，因为所有h2都不是父元素的第一个子元素。
.outer h1:first-of-type：也会匹配到“我是大标题1”和“我是大标题2”，因为它们都是其父元素下使用同种标签的第一个子元素。
.outer h2:first-of-type：会匹配到“我是二标题1”和“我是二标题2”，因为它们都是其父元素下使用同种标签的第一个子元素。

伪元素选择器

选择器 含义
E::before 在E元素之前插入生成的内容
E::after 在E元素之后插入生成的内容

注意点：注入的内容将是E的子元素，它会被置于E元素的任何内容的“前”或“后”。

选择器优先级

从高到低分别是：
（1）、在属性后面使用！important会覆盖页面内任何位置定义的元素样式
（2）、作为style属性写在元素标签上的内联样式
（3）、id选择器
（4）、类选择器
（5）、伪类选择器
（6）、属性选择器
（7）、标签选择器
（8）、通配符选择器
（9）、浏览器自定义
当css规则由多个选择器构成时，可以根据权重做简单的加法运算。

  • 对于选择器中给定的各个ID属性值，加0,1,0,0。
  • 对于选择器中给定的各个类属性值、属性选择或伪类，加0,0,1,0。
  • 对于选择器中给定的各个元素和伪元素，加0,0,0,1。
  • 结合符和通配符选择法对特殊性没有任何贡献。

当两个规则作用于同一个元素的相同属性时，权值高的选择器作用的越具体，因此优先级越高。如果两个选择器规则权值是一样的，则后面的覆盖前面的！
